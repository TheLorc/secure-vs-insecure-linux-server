# Insecure Ubuntu Server (VM)

## Overview
This VM was intentionally configured with weak security settings to demonstrate common Linux vulnerabilities.

## Key Issues
- SSH root login enabled
- Password authentication allowed
- Weak or default passwords
- Firewall disabled 
- Unnecessary services running (e.g Apache2)

## Purpose
To show how misconfigurations can make a Linux server vulnerable to attacks such as brute-force login and unauthorized access.
